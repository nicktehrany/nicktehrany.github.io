Dotfiles

My dotfiles repository, which is more of a continuous project likely to never be done, for managing all my config files for vim, zsh, tmux, git, other dotfiles and utilities I use.

Requirements

  • the default profile contains polybar configs, however these won’t be useful until it is installed, can build it from sources or check if the os repo has it. Therefore if you do not want polybar no need to install it.

Installation

git clone https://github.com/nicktehrany/dotfiles
cd dotfiles

# For installing the default profile
sudo ./install-profile default

Check the .travis.yml for packages that will be installed, if not already installed. All existing links or config files will be overwritten! (Check default-conf.yaml for all links)

Next, to set the default konsole theme, open the konsole menu (Ctrl + Shift + m), under settings select manage profiles, then select blue_default and set it as default. Lastly, to have the status bar of tmux work correctly run inside tmux prefix + I to install required plugins via tpm, followed by reloading tmux with prefix + R.

Next we have to install a font to be able to use nerd font icons in vim. I use the Hack Regular Nerd Font Mono for this. Just have to download and install it, devicons should then directly work since the font is already set in the konsole config (if using a different font just change the font in the konsole config).

Running polybar

I’m using gnome therefore I use the Hide Top Bar extension for hiding the top bar so that polybar can run on top of it. Then to enable the polybar run

polybar/launch.sh

I’m still working on finishing the polybar and adding something for automatically starting polybar

Installing full profile with VSCode

My vscode configs are in the full profile, as I currently do not really use it, but this can be installed with

sudo ./install-profile full

When installing vscode from the full profile with the vscode-conf.yaml, only config files will be linked, installing of vscode and extensions still needs to be done (as this takes quite some time I left it out). But this can easily be run with the vscode installer script

sudo ./vscode/install.sh

Commands

In order to update submodules if their remote repository changes.

git submodule update --remote --merge
